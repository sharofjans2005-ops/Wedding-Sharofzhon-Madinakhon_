# GitHub Pages - Инструкция по развертыванию

## 📱 Что такое GitHub Pages?

GitHub Pages — это бесплатный веб-хостинг для статических сайтов прямо из GitHub репозитория. Идеально подходит для свадебных сайтов!

## 🚀 Развертывание в 5 минут

### Способ 1: Через GitHub Web Interface (Самый простой)

#### Шаг 1: Создайте репозиторий
1. Перейдите на https://github.com/
2. Нажмите `New repository`
3. Назовите репозиторий: `username.github.io`
   - Замените `username` на ваш ник GitHub
4. Выберите "Public"
5. Нажмите "Create repository"

#### Шаг 2: Загрузьте файлы
1. Нажмите "Add file" → "Upload files"
2. Загрузьте все файлы из папки `свабьда`:
   - index.html
   - style.css
   - script.js
   - assets/ (вся папка)
   - README.md
   - Остальные файлы

3. Нажмите "Commit changes"

#### Шаг 3: Готово!
Сайт будет доступен по адресу: `https://username.github.io`

**Важно:** Может потребоваться 1-2 минуты на загрузку.

---

### Способ 2: Через Git (Для продвинутых)

#### Установите Git
Скачайте с https://git-scm.com/

#### Откройте PowerShell в папке проекта

```powershell
# Инициализируем Git
git init

# Добавляем все файлы
git add .

# Первый коммит
git commit -m "Initial commit: Wedding website"

# Добавляем удаленный репозиторий (замените на свой)
git remote add origin https://github.com/username/username.github.io.git

# Отправляем на GitHub (требует авторизации)
git push -u origin main
```

#### Если возникает ошибка с ветвью
```powershell
git branch -M main
git push -u origin main
```

## ✅ После загрузки

### Проверьте сайт
1. Откройте `https://username.github.io` в браузере
2. Страница должна загрузиться за 1-2 минуты
3. Если белая страница — обновите (Ctrl+F5)

### Если сайт не загружается
1. Проверьте имя репозитория (`username.github.io`)
2. Убедитесь, что репозиторий публичный
3. Проверьте, что файл `index.html` в корне
4. Откройте вкладку Settings → Pages
5. Убедитесь, что "Source" установлен на "main branch"

## 🎵 Музыка на GitHub Pages

### Добавление музыки
1. Скачайте MP3 файл (Pixabay.com)
2. Поместите в папку `assets/music/`
3. Переименуйте на `song.mp3`
4. Загрузите на GitHub

**Максимальный размер:** GitHub имеет лимит 100 МБ на файл, 2 ГБ на репозиторий.

### Если файл слишком большой
- Используйте сжатие: https://cloudconvert.com/mp3-compress
- Используйте низкую битрейт (128 кбит/с)
- Обрежьте музыку онлайн: https://www.mp3cut.net/

## 📝 Обновление сайта

### Через Web Interface
1. Откройте файл на GitHub
2. Нажмите карандаш (Edit)
3. Внесите изменения
4. Нажмите "Commit changes"

### Через Git
```powershell
# Внесите изменения в файлы локально

# Добавьте изменения
git add .

# Коммитьте
git commit -m "Update wedding details"

# Отправьте
git push
```

## 🔒 Приватность

### Хотите скрыть сайт от поисковиков?
Добавьте в `index.html` в `<head>`:
```html
<meta name="robots" content="noindex, nofollow">
```

### Защита паролем (продвинутая функция)
Это требует JavaScript или вспомогательного сервиса, так что будет сложнее на GitHub Pages.

## 🎯 Продвинутые параметры

### Собственный домен
1. Купите домен (например, на reg.ru, godaddy.com)
2. В Settings → Pages → Custom domain
3. Введите свой домен
4. Следуйте инструкциям по настройке DNS

### Ограничение доступа (требует Pro)
GitHub Pages не поддерживает аутентификацию нативно, но можно использовать сервисы типа Vercel или Netlify.

## 🆘 Часто возникающие проблемы

| Проблема | Решение |
|----------|---------|
| Белая страница | Обновите кэш (Ctrl+Shift+Delete), подождите 2 мин |
| "Page not found" | Проверьте имя репозитория (username.github.io) |
| CSS не загружается | Проверьте пути в HTML (должны быть относительные) |
| Изображения не видны | Убедитесь, что пути правильные (assets/images/...) |
| Музыка не работает | Убедитесь, что файл в assets/music/song.mp3 |
