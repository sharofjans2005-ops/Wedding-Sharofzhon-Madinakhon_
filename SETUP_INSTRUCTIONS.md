## 📤 Публикация на GitHub Pages

### Способ 1: Через Web (самый простой)

#### 1. Создайте репозиторий на GitHub
1. Перейдите на https://github.com
2. Нажмите **New repository**
3. Назовите: `username.github.io` (замените `username` на ваш ник)
4. Выберите "Public"
5. Нажмите "Create repository"

#### 2. Загрузьте файлы
1. Нажмите **Add file** → **Upload files**
2. Загрузьте все файлы:
   - index.html
   - style.css
   - script.js
   - assets/ (вся папка с music и images)
   - README.md
3. Нажмите **Commit changes**

#### 3. Готово!
Сайт будет доступен по: `https://username.github.io`

### Способ 2: Через Git (для обновлений)

```powershell
# Инициализируем Git (если еще не инициализирован)
git init

# Добавляем все файлы
git add .

# Первый коммит
git commit -m "Wedding website launch"

# Добавляем удаленный репозиторий
git remote add origin https://github.com/username/username.github.io.git

# Отправляем на GitHub
git push -u origin main
```

**Если возникает ошибка с веткой:**
```powershell
git branch -M main
git push -u origin main
```

### Проверка публикации
1. Откройте `https://username.github.io`
2. Страница загружается 1-2 минуты
3. Если белая страница — обновите браузер (Ctrl+Shift+Delete очистить кэш)

## ❓ Часто возникающие вопросы

### В: Музыка не воспроизводится
**О:** Убедитесь, что:
1. Файл находится в `assets/music/song.mp3`
2. Это MP3 файл (не WAV, OGG и т.д.)
3. Размер файла не больше 10 МБ
4. Откройте консоль браузера (F12) и посмотрите ошибки

### В: Сайт не загружается на GitHub Pages
**О:** Проверьте:
1. Имя репозитория: `username.github.io` (точно!)
2. Репозиторий публичный
3. Файл `index.html` в корне репозитория
4. Подождите 1-2 минуты после загрузки
5. Откройте Settings → Pages и убедитесь, что источник установлен

### В: Как обновить сайт после публикации?
**О:** Через Git:
```powershell
git add .
git commit -m "Update wedding details"
git push
```

### В: Могу ли я использовать свой домен вместо github.io?
**О:** Да! Settings → Pages → Custom domain (требуется настройка DNS)

### В: Как защитить сайт паролем?
**О:** GitHub Pages не поддерживает пароли нативно. Используйте Cloudflare или другие сервисы.

### В: Сайт медленно загружается
**О:** Проверьте размер музыкального файла. Оптимально до 5 МБ.
